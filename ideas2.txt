IDEILE PREZENTĂRII — câte o idee per slide
==========================================

1. Titlu — Două subiecte practice: decizia cloud (Claude) vs local/open (Ollama) și memoria persistentă via plugin MCP (total-recall).

2. Ce iei din sesiune — 5 livrabile concrete: ce e Ollama, Claude vs Ollama, integrarea Ollama cu Claude/Copilot/Gemini, cum funcționează total-recall, cum îl instalezi.

3. Agendă — 65 min: 25 Claude vs Ollama, 25 Total Recall, 10 Q&A (cu plan de tăiere la 45 min în note).

4. TEMA 1 — Firul roșu: AI-ul cloud a devenit scump și cu lock-in; Ollama construiește alternativa open.

5. Ce este Ollama — Runtime open-source pentru LLM-uri locale ($0/token, zero date în cloud) + proxy :cloud pentru modele proprietare/mari; 100M+ descărcări Docker Hub, 176k stars GitHub.

6. Ollama · Ce face — Descarcă/rulează modele local, proxy :cloud, API REST pe localhost:11434 compatibilă atât OpenAI cât și Anthropic (de asta merge `ollama launch claude`).

7. Ollama · Instalare — O singură comandă: `ollama run llama3.2`; modelele proprietare cer `ollama signin` + suffix `:cloud`.

8. Unde sunt Claude/Gemini/GPT — Local doar open-weight; Gemini doar ca :cloud, GPT doar ca gpt-oss, Claude nu există deloc în Ollama (doar imitații de evitat).

9. Cum funcționează intern — Quantizarea Q4 comprimă modelul de ~8× (70B din 280GB în 48GB VRAM, ~2–5% pierdere); pipeline: API REST → model manager (GGUF) → llama.cpp → GPU/CPU.

10. ollama list real — Pe un laptop obișnuit (MX450 2GB) modelele mari descărcate sunt inutilizabile; coloana SIZE spune totul.

11. Concluzia practică — 2GB VRAM = totul pe CPU; doar modelele de 6–7GB intră în 16GB RAM la 3–8 tok/s; pe acest hardware Claude API rămâne alegerea corectă.

12. eGPU (Bonus) — Singura upgrade-cale pe laptop: eGPU Thunderbolt 4 (~3.500 lei cu RTX 3060 12GB) → 7–13B la 20–50 tok/s; banda TB4 nu contează, VRAM-ul da.

13. GLM-5.2 / Kimi-K2.7 — Modelele :cloud din ollama list NU sunt locale: sunt API-uri externe proxiate (servere Zhipu/Moonshot din China).

14. Mecanica proxy-ului — SIZE=— înseamnă zero GGUF local; avantajul e interfața unică CLI + endpoint-uri OpenAI și Anthropic-compatibile pentru orice model :cloud.

15. Cine sunt cele două — GLM-5.2: aproape de Opus 4.8 pe Terminal-Bench (81.0 vs 85.0) la preț ~12–20× mai mic, MIT, dar română slabă; Kimi-K2.7-Code: specializat pe cod.

16. Regula GDPR — Serverele sunt în China (fără decizie de adecvare); nu trimite date personale, IP proprietar, credențiale; nici Claude nu e „gratuit" GDPR — pentru rezidență EU: Bedrock/Vertex.

17. Regula scurtă — Cod ne-sensibil + buget → :cloud chinezesc; română, date sensibile, raționament critic → Claude.

18. Comparație directă — Nu „care e mai bun", ci ce optimizezi: inteligență maximă (Claude) vs suveranitate totală (Ollama); TTFT local ~120ms vs ~650ms pe API.

19. Cele 5 axe — Claude câștigă la calitate; Ollama la cost/token, confidențialitate, latență; integrarea Claude Code e nativă la ambele.

20. Costul real — Un agentic developer arde ~$400/lună pe API → RTX 4090 se amortizează în ~5 luni; forcing function: Copilot a mutat Opus la 27× multiplicator (iun 2026) — „era AI-ului cloud ieftin s-a terminat".

21. Costul · Claude API — Sonnet: $3/$15 per MTok; sesiune agentică intensă ≈ $18/zi ≈ $400/lună.

22. Costul · Hardware local — RTX 4090 $2.000 (70B, ROI ~5-6 luni), RTX 3080 SH $600 (7-8B, ROI 1-2 luni), CPU-only gratuit dar lent.

23. Găsește-ți tier-ul — Light ($30/lună) → rămâi pe API; daily ($100) → ~6 luni amortizare; agentic ($400) → ~5 luni; Sonnet 5 introductiv $2/$10 schimbă calculul temporar.

24. Confidențialitate — Cu Ollama promptul nu părăsește NICIODATĂ mașina (merge air-gapped); pentru finance/healthcare/NDA e singura opțiune legală. Cu Claude API datele merg la Anthropic (US).

25. Practic cu Claude — Codul și secretele ajung pe servere externe; DPA există dar datele ies din EU; contractele enterprise pot interzice.

26. Ollama local — Zero egress, rețele izolate, datele clientului rămân pe mașină.

27. Integrarea cu Claude Code — Gotcha „subscription hijack": cu abonament Max/Pro, `ANTHROPIC_API_KEY=""` face cererile să plece silențios pe OAuth la Anthropic; fix: doar `ANTHROPIC_AUTH_TOKEN=ollama` + verifică cu /status.

28. Metoda oficială — `ollama launch claude --model <model>` pornește Claude Code cu Ollama ca backend, nativ, fără proxy.

29. Ce face comanda — Setează automat env vars; distincția cheie: formatul e Anthropic, inteligența e modelul Ollama — clientul „crede" că vorbește cu Anthropic.

30. Metoda manuală — Export ANTHROPIC_BASE_URL + AUTH_TOKEN sau permanent în ~/.claude/settings.json.

31. Limitări — Tool calling și extended thinking (basic) merg; prompt caching NU; calitatea = calitatea modelului local.

32. Otari (Bonus) — Gateway Anthropic-compatibil de la Mozilla.ai: același truc BASE_URL dar cu buget enforcement ÎNAINTE de request, chei virtuale, spend tracking, 40+ provideri.

33. Otari · Pornire — Un docker run + 3 env vars; are sens pentru echipe care vor control central de buget fără să dea cheia API brută developerilor.

34. Ghid de decizie — Arbore de decizie printabil în handout; testează cu `ollama launch claude` înainte de decizia de producție.

35. Agent Claude vs LangChain — Niveluri diferite de stivă: model+harness vs framework model-agnostic; diferența e cine deține loop-ul, nu unde rulează.

36. TEMA 2 — Puntea: ai ales runtime-ul; acum fă-l să țină minte cine ești — altfel fiecare sesiune pornește de la zero.

37. Problema — Claude uită tot după sesiune: reexplici contextul, preferințele, deciziile; cu cât lucrezi mai mult, cu atât pierzi mai mult timp.

38. Cine mai vrea asta — Mozilla.ai a lansat cq (shared agent learning, 1.2k stars) — complementar, nu concurent.

39. Demo cu/fără memorie — Aceeași întrebare, două sesiuni: fără plugin „nu am context"; cu plugin recall_memory găsește decizia PostgreSQL — payoff vizibil în 20 de secunde.

40. Soluția: total-recall — Plugin Claude Code care dă AI-ului memorie persistentă, căutabilă, între sesiuni.

41. Ce este — Plugin din marketplace + MCP server cu 17 unelte + vault de fișiere Markdown local + hook-uri automate + skill dedicat.

42. Ce NU este — Nu trimite date în cloud, nu e bază de date opacă (fiecare memorie = fișier .md citibil), nu suprascrie contextul — injectează.

43. Structura pe disc — Zero DB: memoria e un folder de .md-uri pe care le citești, editezi, versionezi cu git, deschizi în Obsidian.

44. Anatomia memoriei — Fișier .md cu frontmatter YAML (title, tags, importanceScore, date) + Executive Summary.

45. Arhitectura — Motor complet de căutare hibridă (TF-IDF + vector + curbă de uitare) în ~24 fișiere TypeScript cu O SINGURĂ dependență obligatorie (MCP SDK).

46. De ce algoritmi proprii — Securitate (fără CVE-uri din dependențe), scoring coerent, determinism (zero apeluri LLM, merge offline), auditabilitate.

47. Dual Vault — Un simplu tag `org` mută memoria din vault personal în vault de echipă (sync git), cu filtru de confidențialitate fail-closed.

48. Filtrul de confidențialitate — Blochează secrete după tipare (PEM, sk-, ghp_, AKIA, JWT), secrete etichetate, email-uri; dacă nu poate analiza → NU face push.

49. Cele 17 unelte MCP — CRUD + căutare + întreținere în limbaj natural — nu înveți nume de unelte; harta pe 4 cadrane: scrie / citește / listează / întreține.

50. Algoritmul de căutare — Memoria uită ca un om: curba Ebbinghaus (1885) în cod; memoriile neaccesate se estompează, fiecare accesare le reîmprospătează cu +20%.

51. Esența algoritmului — TF-IDF (citește doar indexul) × decay exponențial guvernat de importanceScore; feedback prin confirm/flag (±10%).

52. Embeddings (opțional) — Lazy-load, complet locale (MiniLM sau bge-m3 via Ollama); fără ele pluginul face graceful downgrade la TF-IDF; multilingual EN↔RO prin expansiune de tokeni.

53. Demo EN↔RO — Stochezi în engleză, întrebi în română — „decizie"→„decision" găsește memoria; climaxul temei 2.

54. Hook-urile — PreCompact salvează automat learnings-urile sesiunii ÎNAINTE ca contextul să fie șters; rulează și pe Copilot CLI și Gemini CLI.

55. Pe Copilot/Gemini — Side-effects (sync, index) rulează normal; limitarea: clienții ignoră stdout → fără context injection automat, dar MCP-ul rămâne interogabil.

56. SessionStart — 4 pași: pull org-vault, build index, injectare index + open-questions în context — la fiecare sesiune, fără să ceri.

57. PostToolUse — După store/update/delete: verifică tag org, aplică privacy filter, push pe git, rebuild cache.

58. PreCompact/SessionEnd — Extrage 0–3 learnings din transcript și le scrie direct ca .md; SessionEnd face cleanup și flush.

59. Instalare — Același plugin, 4 clienți (Claude Code, Copilot, Gemini, standalone) cu un singur install.sh.

60. Build — git clone marketplace + npm install + npm run build.

61. Per client — claude plugin install / install.sh --copilot / --gemini / --standalone.

62. Org vault opțional — Dezactivat implicit; personal-vault merge 100% din prima, fără git; org doar pentru memorie de echipă.

63. Utilizare — Totul în limbaj natural: „reamintește-mi…", „reține că…", „arată-mi memoriile de…".

64. Compatibilitate — 3 niveluri: Full (Claude Code), Parțial (Copilot/Gemini — fără auto-injecție), Manual (Codex — doar MCP).

65-66. Matricea — MCP merge peste tot; hooks pe 3 clienți; context injection și skills doar în Claude Code.

67. Obsidian & cq — Vault-urile se deschid nativ în Obsidian; cq acoperă 7 host-uri dar fără context injection — obiecte diferite (memorie de context vs knowledge units).

68. Sinteza — 3 lucruri de făcut mâine: instalează Ollama, rulează `ollama launch claude --model gemma3:4b`, adaugă total-recall.

69. Sinteza Claude vs Ollama — Decizia cheie: confidențialitate > calitate > cost.

70. Sinteza Total Recall + Mozilla.ai — Kit de suveranitate: llamafile + any-llm + Otari + cq = răspunsul open-source la scumpirea AI-ului cloud.

71. Întrebări deschise — Integrare în echipă, modele testate pe hardware real, combinarea Ollama+Claude, backup vault.

72-77. Resurse — Link-uri: Ollama, Claude API (prețuri și modele curente), integrare, ecosistem Mozilla.ai, Total Recall, handout-uri printabile.

78. Q&A — 10 minute; tehnici de regie: echo chamber, seeded first question, posse.
