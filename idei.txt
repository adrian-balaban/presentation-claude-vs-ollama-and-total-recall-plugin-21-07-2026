SLIDE 5 — Ce este Claude?
- Claude = LLM-uri remote proprietare (Fable, Sonnet, Opus, Haiku) + agent local proprietar (Claude Code CLI)
- Modelele rulează exclusiv pe serverele Anthropic sau in cloud (AWS Bedrock...)
- Agentul local Claude Code e pe mașina ta
- Comenzi: `claude`, `/status`, `/models`, `/effort`, ...
- Interfata de la https://claude.ai

SLIDE 6 — Ce este Ollama?
- Tool open-source pentru rularea LLM-urilor local, pe propriul hardware
- Zero conexiune la internet, zero cost per token (pentru modele locale)
- Pentru Modele prea mari: Ollama face proxy la cloud-ul Ollama (nu rămân locale)
- Interfata de la https://ollma.ai si https://docs.ollama.com si github
- WOW: 100M+ descărcări pe Docker Hub, 176k stars pe GitHub

SLIDE 9 — Avantajul Ollama: o interfață, modele locale și remote, multiple integrări
- Aceeași CLI pentru toate modelele (`ollama run <model>:cloud`)
- Endpoint-uri compatibile OpenAI, Anthropic etc.
- Lista completă integrări: docs.ollama.com/integrations

SLIDE 10 — Ce este Ollama? · Modelele disponibile

SLIDE 15 — Comparație directă: Claude vs Ollama
- Nu e „care e mai bun", ci ce optimizezi: inteligență maximă vs suveranitate totală vs cost vs viteza raspuns
- WOW metric: TTFT ~120ms (Ollama GPU local model 8B) vs ~650ms (Claude API) — de ~5x mai rapid local
- Măsurare: scripts/ttft.sh pe ambele endpoint-uri
- Avertisment: pe CPU-only (demo laptop) = 800–900ms, nu 120ms
- Fallback: înregistrări asciinema (casts/slide-15.cast)

SLIDE 16 — GDPR: unde ajung datele
- GLM-5.2, Kimi-K2.7-Code :cloud → servere în China, fără GDPR
- Claude: pentru rezidență EU, necesită AWS Bedrock / Vertex AI pe regiuni EU

SLIDE 17 — Costul real: Claude API vs Ollama
- WOW: ~$400/lună (agentic developer) → RTX 4090 $2.000 se amortizează în ~5 luni
- Indie/light: ~$30/lună → amortizare durează ani
- Daily driver: ~$100/lună → ~6 luni (RTX 3080 SH ~$600)

SLIDE 18 — Confidențialitate și date: diferența crucială
- WOW: Ollama local = promptul și datele nu părăsesc niciodată mașina, merge air-gapped
- Finance/healthcare/NDA: Ollama local e singura opțiune legală
- Fluxul Claude API: HTTPS → Anthropic US → procesare, logging; training: implicit NU, dar citiți ToS

SLIDE 19 — Confidențialitate și date · Practic, cu Claude
- Codul și secretele din prompt ajung la Anthropic
- GDPR: DPA disponibil, dar datele ies din EU
- Ollama local: zero egress; :cloud = datele pleacă la furnizor, ca orice API

SLIDE 20 — Ollama cu Claude Code · Limitări
- Suportă tool calling și extended thinking (budget_tokens acceptat dar neaplicat)
- Nu suportă prompt caching
- Calitatea = calitatea modelului ales
